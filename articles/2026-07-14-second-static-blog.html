从零开始发布我的第二个静态博客

这是我的第二篇博客文章，用来测试 Markdown 写作、自动生成文章页和 GitHub Pages 发布流程。

今天我成功发出了第二篇文章。现在写博客不用直接改 index.html，只需要在 posts 文件夹里新建 Markdown 文件。

我这次学到了什么

  • 文章源文件应该放在 posts 文件夹
  • 文件名最好写清楚，例如 2026-07-14-second-static-blog.md
  • 修改文章后要运行 .\tools\build-blog.ps1
  • 最后用 git add .、git commit、git push 发布
